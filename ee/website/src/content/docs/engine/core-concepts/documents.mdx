# Documents

A **document** is the central object in CloudPDF. It wraps the underlying file,
its rendered pages, annotations, and the access policy that governs who can see
it.

## Creating a document

Upload a file from your server. CloudPDF processes it asynchronously and
returns a document record immediately.

```ts
const doc = await cloudpdf.documents.create({
  file: './contracts/acme-2026.pdf',
  metadata: { customerId: 'cus_8812', kind: 'contract' },
});

console.log(doc.id); // doc_3kZ…
```

The optional `metadata` object is yours to use — filter and search on it later
from the dashboard or the list API.

## Listing and filtering

```ts
const { data, nextCursor } = await cloudpdf.documents.list({
  filter: { 'metadata.kind': 'contract' },
  limit: 25,
});
```

Pagination is cursor-based. Pass `nextCursor` back in to fetch the next page.

## Document status

A freshly uploaded document moves through a short lifecycle:

- `processing` — the file is being parsed and rendered.
- `ready` — pages are available and the document can be signed.
- `failed` — the file could not be processed (corrupt or unsupported).

<Callout type="info">
  Subscribe to the `document.ready` webhook to be notified the moment a document
  becomes available, instead of polling.
</Callout>

## Deleting a document

Deletion is permanent and removes rendered pages, annotations, and audit
history.

```ts
await cloudpdf.documents.delete('doc_3kZ…');
```

<Callout type="warn">
  Deleting a document immediately invalidates every signed URL pointing at it.
  Existing viewers will lose access on their next request.
</Callout>
