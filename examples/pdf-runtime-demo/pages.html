Engine v3 page reorder browser demo

Drives doc.pages.list() → single-page move() (last → front) → multi-page contiguous-block move() against /sample.pdf, using createLocalEngineWithWorker from @embedpdf/engine with the WASM runtime in a Web Worker. The output below shows every page's (pageObjectNumber, pageIndex, revision.generation) tuple before and after each move. Locked invariants: per-page revision.generation values are unchanged across page moves (so weak annotation refs survive a reorder), the PON set is preserved, and pageIndex is dense in [0, count - 1].
